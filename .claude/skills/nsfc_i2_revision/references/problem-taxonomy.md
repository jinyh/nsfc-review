# NSFC 申请书问题分类体系

## 概述

本分类体系用于诊断评审专家意见中指出的问题类型，指导修改策略和文献检索。

**5 大类 + 15 小类**：
1. 科学问题类（3 小类）
2. 交叉融合类（3 小类）
3. 方法路线类（3 小类）
4. 验证方案类（3 小类）
5. 形式与表达类（3 小类）

---

## 1. 科学问题类

### 1.1 伪问题

**定义**：专家质疑研究问题不是真实的科学鸿沟，或已被充分解决。

**诊断关键词**：
- "这个问题已经解决"
- "不是真正的科学问题"
- "缺乏科学意义"
- "问题不成立"
- "研究动机不足"

**修改策略**：
- 补充文献综述，说明现有方法的局限性
- 引用最新综述文章，证明问题仍是 open problem
- 明确指出与已有工作的差异（时空尺度、精度要求、应用场景等）

**文献检索关键词**：
- `[核心概念] + "review" + "challenges"`
- `[核心概念] + "open problems"`
- `[核心概念] + "future directions"`

**示例**：
- **专家意见**："AI 预测蛋白质结构已经被 AlphaFold 解决，研究意义不大。"
- **问题诊断**：伪问题（专家认为问题已解决）
- **修改策略**：补充 AlphaFold 的局限性（如动态构象、多体复合物、膜蛋白等），引用 Nature 综述说明仍存在的挑战
- **检索关键词**：`"protein structure prediction" + "AlphaFold" + "limitations"`

---

### 1.2 可证伪性不足

**定义**：研究问题缺乏明确的度量指标、边界条件或失败判据，无法验证假设的真伪。

**诊断关键词**：
- "如何验证"
- "缺乏度量指标"
- "边界条件不清"
- "失败判据是什么"
- "如何判断成功"

**修改策略**：
- 明确定义成功标准（定量指标 + 阈值）
- 说明失败边界（方法不适用的场景）
- 设计对照实验或消融实验

**文献检索关键词**：
- `[应用领域] + "evaluation metrics"`
- `[应用领域] + "benchmark"`
- `[应用领域] + "validation criteria"`

**示例**：
- **专家意见**："如何验证你的方法真的捕捉到了物理规律，而不是数据拟合？"
- **问题诊断**：可证伪性不足（缺乏验证标准）
- **修改策略**：增加物理一致性检验（如能量守恒、对称性、外推能力），引用 physics-informed ML 文献中的验证方法
- **检索关键词**：`"physics-informed machine learning" + "validation" + "physical consistency"`

---

### 1.3 研究现状不充分

**定义**：文献综述不足，未充分说明与已有工作的差异，或遗漏重要相关工作。

**诊断关键词**：
- "文献综述不足"
- "遗漏了重要工作"
- "与XX工作的差异不清"
- "缺乏对比分析"
- "研究现状不充分"

**修改策略**：
- 补充遗漏的重要文献（尤其是近 3 年的工作）
- 增加对比表格，明确与已有工作的差异
- 说明本研究的独特贡献

**文献检索关键词**：
- `[核心概念] + "recent advances"`
- `[核心概念] + "state-of-the-art"`
- `[核心概念] + [年份]`

**示例**：
- **专家意见**："申请书未提及 DeepMind 2024 年的工作，该工作与本研究高度相关。"
- **问题诊断**：研究现状不充分（遗漏重要文献）
- **修改策略**：补充 DeepMind 2024 工作，说明其局限性（如仅适用于特定体系），强调本研究的差异（如多尺度耦合、可解释性）
- **检索关键词**：`"DeepMind" + [核心概念] + "2024"`

---

## 2. 交叉融合类

### 2.1 两张皮

**定义**：AI 与领域科学缺乏深度融合，仅是简单叠加或并列，未体现交叉优势。

**诊断关键词**：
- "两张皮"
- "缺乏深度融合"
- "AI 只是工具"
- "未体现交叉优势"
- "领域知识未嵌入"

**修改策略**：
- 说明 AI 如何改变领域科学的研究范式（而非仅提高效率）
- 强调双向融合：AI 嵌入领域约束 + 领域问题驱动 AI 创新
- 增加融合机制的技术细节

**文献检索关键词**：
- `[学科交叉方向] + "interdisciplinary"`
- `"AI for science" + [领域]`
- `"physics-informed" / "chemistry-informed" / "biology-informed"`

**示例**：
- **专家意见**："申请书中 AI 只是用来拟合数据，未体现与物理的深度融合。"
- **问题诊断**：两张皮（AI 与物理缺乏融合）
- **修改策略**：增加物理约束嵌入机制（如损失函数中加入物理方程、网络结构体现对称性），引用 physics-informed neural networks 文献
- **检索关键词**：`"physics-informed neural networks" + "physical constraints"`

---

### 2.2 领域知识嵌入不足

**定义**：未说明如何将领域约束（物理定律、化学规则、生物机制等）嵌入 AI 模型。

**诊断关键词**：
- "如何嵌入领域知识"
- "物理约束在哪里"
- "先验知识如何利用"
- "缺乏领域指导"

**修改策略**：
- 明确领域知识嵌入的三个层次：
  - 数据层：特征工程、数据增强
  - 模型层：网络结构、损失函数
  - 推理层：后处理、约束优化
- 提供具体技术方案（如 Lagrangian 乘子法、软约束、硬约束）

**文献检索关键词**：
- `"domain knowledge embedding" + [领域]`
- `"prior knowledge" + "neural networks"`
- `"constraint-based learning"`

**示例**：
- **专家意见**："如何将化学反应的选择性规则嵌入模型？"
- **问题诊断**：领域知识嵌入不足
- **修改策略**：增加选择性规则嵌入方案（如在损失函数中加入选择性惩罚项，或在网络输出层加入规则过滤），引用 chemistry-informed ML 文献
- **检索关键词**：`"chemistry-informed machine learning" + "selectivity rules"`

---

### 2.3 多尺度耦合缺失

**定义**：未处理不同学科的时空尺度差异，或多尺度耦合机制不清晰。

**诊断关键词**：
- "多尺度如何耦合"
- "时空尺度差异"
- "跨尺度建模"
- "粗粒化与精细化"

**修改策略**：
- 明确各尺度的建模方法（如量子力学、分子动力学、连续介质力学）
- 说明尺度间的信息传递机制（如粗粒化、上采样、多分辨率网络）
- 引用多尺度建模文献

**文献检索关键词**：
- `"multiscale modeling" + [领域]`
- `"coarse-graining" + "machine learning"`
- `"scale bridging"`

**示例**：
- **专家意见**："如何将原子尺度的量子计算与宏观尺度的流体力学耦合？"
- **问题诊断**：多尺度耦合缺失
- **修改策略**：增加多尺度耦合方案（如使用 graph neural networks 在原子尺度建模，通过粗粒化映射到连续场，再用 CFD 求解宏观流动），引用多尺度 AI 文献
- **检索关键词**：`"multiscale machine learning" + "quantum to continuum"`

---

## 3. 方法路线类

### 3.1 必要性论证不足

**定义**：未说明为何传统方法不够，或为何需要 AI 方法。

**诊断关键词**：
- "为什么需要 AI"
- "传统方法为何不够"
- "必要性不足"
- "动机不清"

**修改策略**：
- 明确传统方法的瓶颈（计算成本、精度、泛化能力等）
- 说明 AI 方法的独特优势（如高维非线性建模、数据驱动发现）
- 引用对比文献

**文献检索关键词**：
- `[传统方法] + "limitations"`
- `[AI 方法] + "advantages over" + [传统方法]`

**示例**：
- **专家意见**："为什么不用传统的密度泛函理论（DFT），而要用神经网络？"
- **问题诊断**：必要性论证不足
- **修改策略**：说明 DFT 的计算成本瓶颈（如大体系、长时间模拟），强调神经网络的加速优势（如 DeepMD 可达 6 个数量级加速），引用对比文献
- **检索关键词**：`"neural network potentials" + "vs DFT" + "computational cost"`

---

### 3.2 创新性不足

**定义**：与已有方法差异不明显，或创新点不清晰。

**诊断关键词**：
- "创新性不足"
- "与XX方法的差异"
- "新在哪里"
- "贡献不清"

**修改策略**：
- 明确列出创新点（方法创新、应用创新、理论创新）
- 与已有方法对比，突出差异
- 说明创新点的科学价值

**文献检索关键词**：
- `[已有方法] + "limitations"`
- `[核心概念] + "novel approach"`

**示例**：
- **专家意见**："你的图神经网络与 SchNet 的差异是什么？"
- **问题诊断**：创新性不足（与已有方法差异不清）
- **修改策略**：明确差异（如 SchNet 仅考虑局部环境，本研究引入长程相互作用；SchNet 不可解释，本研究增加注意力机制），引用 SchNet 论文并对比
- **检索关键词**：`"SchNet" + "limitations" + "long-range interactions"`

---

### 3.3 可解释性缺失

**定义**：黑箱模型，缺乏机理解释，难以获得科学洞见。

**诊断关键词**：
- "黑箱模型"
- "缺乏可解释性"
- "如何理解机理"
- "物理意义不清"

**修改策略**：
- 增加可解释性方案（如注意力可视化、特征重要性分析、符号回归）
- 说明如何从模型中提取科学洞见
- 引用可解释 AI 文献

**文献检索关键词**：
- `"interpretable machine learning" + [领域]`
- `"explainable AI" + "scientific discovery"`
- `"attention mechanism" + "physical interpretation"`

**示例**：
- **专家意见**："神经网络是黑箱，如何从中获得化学反应机理？"
- **问题诊断**：可解释性缺失
- **修改策略**：增加可解释性方案（如使用注意力机制识别关键原子，使用符号回归提取反应速率方程），引用可解释化学 AI 文献
- **检索关键词**：`"interpretable machine learning" + "chemical reaction mechanisms"`

---

## 4. 验证方案类

### 4.1 指标体系不完整

**定义**：仅有 ML 指标（如 RMSE、R²），缺乏领域指标（如物理一致性、化学合理性）。

**诊断关键词**：
- "缺乏领域指标"
- "仅有 ML 指标"
- "如何评价物理合理性"
- "验证标准不全"

**修改策略**：
- 补充领域指标（如能量守恒误差、对称性偏差、外推能力）
- 说明各指标的物理意义
- 引用领域验证标准

**文献检索关键词**：
- `[应用领域] + "evaluation metrics"`
- `[应用领域] + "validation criteria"`
- `"physical consistency" + "metrics"`

**示例**：
- **专家意见**："除了 RMSE，如何评价模型的物理合理性？"
- **问题诊断**：指标体系不完整（缺乏物理指标）
- **修改策略**：增加物理指标（如能量守恒误差、力的旋度为零、温度的正定性），引用 physics-informed ML 验证标准
- **检索关键词**：`"physics-informed machine learning" + "validation metrics"`

---

### 4.2 验证层次单一

**定义**：仅有仿真验证或仅有数据验证，缺乏递进式验证（如从简单到复杂、从已知到未知）。

**诊断关键词**：
- "验证层次单一"
- "缺乏递进验证"
- "如何验证泛化能力"
- "实验验证不足"

**修改策略**：
- 设计多层次验证方案：
  - 层次 1：已知体系验证（与理论/实验对比）
  - 层次 2：复杂体系验证（与高精度仿真对比）
  - 层次 3：未知体系预测（与实验验证）
- 说明各层次的目的和意义

**文献检索关键词**：
- `[应用领域] + "validation hierarchy"`
- `[应用领域] + "experimental validation"`

**示例**：
- **专家意见**："仅在仿真数据上验证不够，如何保证在真实体系中有效？"
- **问题诊断**：验证层次单一（缺乏实验验证）
- **修改策略**：增加实验验证计划（如与合作者的实验数据对比，或设计可实验验证的预测），引用类似工作的验证方案
- **检索关键词**：`[应用领域] + "experimental validation" + "machine learning"`

---

### 4.3 失败边界未分析

**定义**：未说明方法的局限性和适用范围，或失败案例分析不足。

**诊断关键词**：
- "局限性是什么"
- "适用范围"
- "失败案例"
- "边界条件"

**修改策略**：
- 明确方法的适用范围（如体系大小、温度范围、化学环境）
- 分析失败案例（如哪些情况下方法不适用）
- 说明失败的原因和改进方向

**文献检索关键词**：
- `[方法名称] + "limitations"`
- `[方法名称] + "failure cases"`

**示例**：
- **专家意见**："你的方法在哪些情况下会失效？"
- **问题诊断**：失败边界未分析
- **修改策略**：增加失败边界分析（如在强关联体系、相变点附近、化学键断裂时方法可能失效），说明原因（如训练数据不足、物理假设不成立），引用类似方法的局限性分析
- **检索关键词**：`[方法名称] + "limitations" + "failure modes"`

---

## 5. 形式与表达类

### 5.1 题目过大

**定义**：研究范围过宽，缺乏聚焦，难以在项目周期内完成。

**诊断关键词**：
- "题目过大"
- "范围过宽"
- "缺乏聚焦"
- "不切实际"

**修改策略**：
- 缩小研究范围（如从"通用方法"改为"特定体系"）
- 明确研究边界（如仅考虑特定条件、特定尺度）
- 说明为何这个范围是合理的

**无需文献检索**

**示例**：
- **专家意见**："题目'AI 驱动的材料设计'过于宽泛，建议聚焦到特定材料体系。"
- **问题诊断**：题目过大
- **修改策略**：缩小范围（如改为"AI 驱动的钙钛矿太阳能电池材料设计"），说明为何选择这个体系（如应用前景、数据可得性）

---

### 5.2 表达不清

**定义**：核心概念、技术路线表述模糊，难以理解。

**诊断关键词**：
- "表达不清"
- "概念模糊"
- "技术路线不清"
- "难以理解"

**修改策略**：
- 明确定义核心概念（如给出数学定义、物理意义）
- 增加技术路线图（流程图、示意图）
- 使用具体例子说明

**无需文献检索**

**示例**：
- **专家意见**："'多尺度耦合'的具体含义不清，建议明确定义。"
- **问题诊断**：表达不清（概念模糊）
- **修改策略**：增加多尺度耦合的定义（如"将原子尺度的量子力学计算与介观尺度的相场模拟通过粗粒化映射耦合"），增加示意图

---

### 5.3 格式问题

**定义**：文献标注、图表、错别字等格式问题。

**诊断关键词**：
- "文献标注不规范"
- "图表不清晰"
- "错别字"
- "格式问题"

**修改策略**：
- 修正文献标注格式（按 NSFC 要求）
- 优化图表（增加标题、坐标轴标签、图例）
- 校对错别字

**无需文献检索**

**示例**：
- **专家意见**："图 2 缺少坐标轴标签，难以理解。"
- **问题诊断**：格式问题（图表不清晰）
- **修改策略**：为图 2 增加坐标轴标签（如"温度 (K)"、"能量 (eV)"）、图例、标题

## 使用指南

### 诊断流程

1. **读取专家意见**：提取关键批评/建议
2. **匹配关键词**：在 15 个小类中查找匹配的诊断关键词
3. **确定问题类型**：选择最匹配的小类
4. **决定是否检索文献**：
   - 类型 1-4（科学问题、交叉融合、方法路线、验证方案）→ 检索文献
   - 类型 5（形式与表达）→ 无需检索
5. **生成修改建议**：根据修改策略和检索结果

### 多问题诊断

若一条专家意见涉及多个问题类型，应：
- 分别诊断每个问题
- 分别生成修改建议
- 在回复文本中综合回应

**示例**：
- **专家意见**："申请书题目过大，且未说明与传统 DFT 方法的差异。"
- **问题诊断**：
  - 问题 1：题目过大（5.1）
  - 问题 2：必要性论证不足（3.1）
- **修改建议**：
  - 针对问题 1：缩小题目范围
  - 针对问题 2：补充 DFT 局限性分析，引用对比文献
