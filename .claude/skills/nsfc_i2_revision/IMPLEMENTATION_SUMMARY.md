## 核心特性

### 1. 问题分类体系（5 大类 + 15 小类）

| 大类 | 小类数量 | 是否检索文献 |
|-----|---------|-------------|
| 科学问题类 | 3 | ✓ |
| 交叉融合类 | 3 | ✓ |
| 方法路线类 | 3 | ✓ |
| 验证方案类 | 3 | ✓ |
| 形式与表达类 | 3 | ✗ |

### 2. 文献检索策略

- **触发条件**：根据问题类型自动决定是否检索
- **检索数量**：每条意见 2-3 篇，总计 10-20 篇
- **检索范围**：Google Scholar、Semantic Scholar、arXiv、领域顶级期刊
- **诚实原则**：若检索受限或未找到相关文献，如实说明，不编造引用

### 3. 回复文本生成

三种回复类型：
1. **对批评的回复**：感谢 → 认可问题 → 说明修改 → 引用文献支撑
2. **对建议的回复**：感谢 → 赞同观点 → 说明实施 → 预期效果
3. **对肯定的回复**：感谢 → 进一步完善 → 目标

### 4. 输出格式

结构化 Markdown 表格，包含 5 列：
- 原始意见
- 问题诊断
- 修改建议
- 回复文本
- 支撑文献

## 与 nsfc_i2_review 的差异化

| 特性 | nsfc_i2_review | nsfc_i2_revision |
|-----|----------------|------------------|
| **视角** | 评审专家（找问题） | 申请人（解决问题） |
| **输入** | 申请书 PDF | 评审意见文件 |
| **输出** | 问题清单 + 改进建议 | 修改建议 + 回复文本 |
| **文献检索** | 无 | 有（10-20 篇） |
| **使用阶段** | 提交前自查 | 收到评审意见后修改 |
| **独立性** | 独立 | 独立（不依赖 nsfc_i2_review） |
