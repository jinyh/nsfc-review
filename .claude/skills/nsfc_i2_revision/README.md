# nsfc_i2_revision Skill 使用说明

## 概述

`nsfc_i2_revision` 是一个用于处理 NSFC 基金申请书评审意见并生成修改响应的 skill。

**核心功能**：
- 解析评审意见文件（Markdown 或 PDF）
- 诊断问题类型（5 大类 + 15 小类）
- 检索相关文献支撑修改建议
- 生成具体、可操作的修改建议
- 撰写礼貌、专业的回复文本模板
- 输出结构化表格报告

## 与 nsfc-review 的区别

| 特性 | nsfc_i2_review | nsfc_i2_revision |
|-----|----------------|------------------|
| **视角** | 评审专家（找问题） | 申请人（解决问题） |
| **输入** | 申请书 PDF | 评审意见文件 |
| **输出** | 问题清单 + 改进建议 | 修改建议 + 回复文本 |
| **文献检索** | 无 | 有（10-20 篇） |
| **使用阶段** | 提交前自查 | 收到评审意见后修改 |

## 使用方法

### 1. 准备评审意见文件

支持两种格式：

**Markdown 格式**（推荐）：
```markdown
# 专家1 评审意见

## 基础科学问题
[专家评价]

## 创新性与科学价值
[专家评价]

## 研究基础与可行性
[专家评价]

## 其他建议
[专家评价]

---

# 专家2 评审意见
[同上结构]
```

**PDF 格式**：
- 直接提供评审意见 PDF 文件路径

### 2. 调用 skill

在 Claude Code 中输入：

```
我收到了评审意见（review_comments_xu.md），请帮我生成修改响应报告。
```

或：

```
请帮我答复专家的评审意见，文件在 /path/to/review_comments.pdf
```

### 3. 获得输出

Skill 将生成一个结构化的 Markdown 表格报告，包含：

| 原始意见 | 问题诊断 | 修改建议 | 回复文本 | 支撑文献 |
|---------|---------|---------|---------|---------|
| [专家原话] | [问题类型 + 分析] | [具体修改方案] | [回复模板] | [文献引用] |

### 4. 使用报告

1. **复制修改建议**：根据"修改建议"列修改申请书
2. **填写页码**：将"第X页"占位符替换为实际页码
3. **复制回复文本**：将"回复文本"列的内容复制到答复文档
4. **人工审核**：检查修改建议和回复文本的准确性和适当性

## 问题分类体系

Skill 使用 5 大类 + 15 小类的问题分类体系：

### 1. 科学问题类
- 1.1 伪问题
- 1.2 可证伪性不足
- 1.3 研究现状不充分

### 2. 交叉融合类
- 2.1 两张皮
- 2.2 领域知识嵌入不足
- 2.3 多尺度耦合缺失

### 3. 方法路线类
- 3.1 必要性论证不足
- 3.2 创新性不足
- 3.3 可解释性缺失

### 4. 验证方案类
- 4.1 指标体系不完整
- 4.2 验证层次单一
- 4.3 失败边界未分析

### 5. 形式与表达类
- 5.1 题目过大
- 5.2 表达不清
- 5.3 格式问题

详见 `references/problem-taxonomy.md`。

## 文献检索策略

| 问题类型 | 是否检索 | 检索目标 |
|---------|---------|---------|
| 科学问题类 | ✓ | 证明问题真实性，找到科学鸿沟的文献支撑 |
| 交叉融合类 | ✓ | 找到融合范式的成功案例和方法论 |
| 方法路线类 | ✓ | 找到同类方法的对比基线和改进方向 |
| 验证方案类 | ✓ | 找到领域公认的验证标准和评价指标 |
| 形式与表达类 | ✗ | 直接给出修改建议 |

**检索数量**：每条需要文献支撑的意见至少检索 2-3 篇相关文献，总计 10-20 篇。

**诚实原则**：若检索受限或未找到直接相关文献，如实说明，不编造引用。

## 注意事项

1. **文献诚实**：不得编造引用，若检索受限需如实说明
2. **人工审核**：生成的修改建议和回复文本需人工审核后使用
3. **页码占位符**：输出中的"第X页"为占位符，需根据实际申请书填写
4. **独立性**：本 skill 不依赖 `nsfc_i2_review`，可独立运行
5. **差异化**：站在申请人视角（解决问题），而非评审专家视角（找问题）

## 测试

使用 `review_comments_xu.md` 作为测试输入：

```
我收到了评审意见（review_comments_xu.md），请帮我生成修改响应报告。
```

预期输出：
- 覆盖 5 位专家的所有意见
- 每条意见都有问题诊断、修改建议、回复文本
- 需要文献支撑的意见都检索了文献
- 输出格式为结构化 Markdown 表格

## 扩展性

未来可扩展的方向：
1. **全局诊断模式**：增加"多位专家共性问题提取"功能
2. **章节式输出**：按申请书章节组织修改建议（需读取申请书 PDF）
3. **优先级排序**：按问题严重程度对修改建议排序
4. **自动修改**：直接生成修改后的申请书文本（需谨慎，建议人工审核）
5. **通用化**：扩展到其他基金类型（重点项目、杰青等）或其他学科方向
